

=== 구조 분석 ===
**📌 프로젝트 유형: 모바일 앱**

이 모바일 앱은 3개의 메인 스크린와 23개의 독립 뷰로 구성되어 있다.

첫 번째 스크린는 '뮤즈미-인트로-screen'이다.
이 '뮤즈미-인트로-screen'은 6행 1열의 뷰로 구성되어 있다.
*   첫 번째 행에는 *   '헤더-그룹(1)-view'가 위치한다.
    -   기능: 웹 페이지 상단에 로고, 메뉴 등 핵심 요소를 담는 영역
*   두 번째 행에는 *   '배너-갤러리-display'가 위치한다.
    -   기능: 여러 개의 배너 이미지를 순환하며 보여주는 출력 영역
    *   이 '배너-갤러리-display'는 다시 1개의 요소로 구성되어 있다.
        *   '네비게이션-view'가 위치한다.
            -   기능: 배너 갤러리의 이전/이후 이동 버튼과 넘버링을 포함하는 영역
            *   이 '네비게이션-view'는 다시 1행 3열의 뷰로 구성되어 있다.
                *   1열에는 '이전-배너-touch-button'가 위치한다.
                    -   기능: 현재 배너에서 이전 배너로 전환하는 버튼
                *   2열에는 '배너-넘버링-display'가 위치한다.
                    -   기능: 현재 보이는 배너의 순서나 총 개수를 표시하는 영역
                *   3열에는 '이후-배너-touch-button'가 위치한다.
                    -   기능: 현재 배너에서 다음 배너로 전환하는 버튼
*   세 번째 행에는 *   '전통-테마-갤러리-view'가 위치한다.
    -   기능: 전통 테마의 제품들을 모아 보여주는 섹션

        *   1행에는 '전통-카테고리-view'가 위치한다.
            -   기능: 전통 테마 내에서 제품 종류별로 분류하는 영역
        *   2행에는 '전통-테마-전체-페이지-입장-touch-button'가 위치한다.
            -   기능: 전통 테마의 모든 제품을 볼 수 있는 페이지로 이동
*   네 번째 행에는 *   '데일리-테마-갤러리-view'가 위치한다.
    -   기능: 데일리 테마의 제품들을 모아 보여주는 섹션
    *   이 '데일리-테마-갤러리-view'는 다시 2행 1열의 뷰로 구성되어 있다.
        *   1행에는 '데일리-카테고리-view'가 위치한다.
            -   기능: 데일리 테마 내에서 제품 종류별로 분류하는 영역
        *   2행에는 '전통-테마-전체-페이지-입장(1)-touch-button'가 위치한다.
            -   기능: 데일리 테마의 모든 제품을 볼 수 있는 페이지로 이동
*   다섯 번째 행에는 *   '공주왕자-테마-갤러리-view'가 위치한다.
    -   기능: 공주왕자 테마의 제품들을 모아 보여주는 섹션
    *   이 '공주왕자-테마-갤러리-view'는 다시 2행 1열의 뷰로 구성되어 있다.
        *   1행에는 '공주왕자-카테고리-view'가 위치한다.
            -   기능: 공주왕자 테마 내에서 제품 종류별로 분류하는 영역

사용자가 `각-테마별-카테고리-view`에서 특정 테마를 선택하면, 해당 테마의 상세 카테고리인 `전통-카테고리-view`, `데일리-카테고리-view`, 또는 `공주왕자-카테고리-view`가 활성화되어 관련 제품을 보여줍니다. 이 카테고리들은 각각 `전통-테마-갤러리-view`, `데일리-테마-갤러리-view`, `공주왕자-테마-갤러리-view`를 통해 실제 제품 갤러리 화면으로 이어집니다.

한편, `헤더-그룹(1)-view`에서 전달된 정보는 `헤더-그룹-컴포넌트-view`를 거쳐 `헤더-그룹(1)(1)-view` 및 `헤더-그룹(1)(1)(1)-view`와 같은 다양한 헤더 컴포넌트들을 구성하며, `tab-menu-view`를 통해 주요 내비게이션 기능을 제공합니다. `네비게이션-view`는 미정의된 기능을 수행하며, `헤더1-view`와 `헤더-그룹(1)(2)-view` 또한 헤더 영역의 일부로 기능합니다.

제품 탐색 흐름에서, `제품-그룹-view`는 제품 관련 화면들을 묶는 역할을 합니다. `필터링-view`는 사용자의 선택에 따라 `제품-갤러리-view`에 표시되는 제품 목록을 동적으로 변경합니다. `제품-갤러리-view`는 `제품-갤러리-요약-및-보기-옵션-view`를 통해 제품 요약 정보와 보기 옵션을 제공하며, `제품-갤러리-4열-view`와 같은 특정 레이아웃으로 제품을 진열합니다.

Part 1에서 전달된 `제품별_데이터-data`와 `제품별_이미지-resource`는 `제품별_데이터들-folder`에 저장되며, 이 데이터는 `구매-정보-view`로 전달되어 특정 제품의 상세 정보를 표시합니다. `구매-정보-view` 내에서는 `금액-소개-view`를 통해 제품 가격 정보를 확인하고, `구먀/대여-결정-여부-view`를 통해 구매 또는 대여 여부를 선택할 수 있습니다. 또한 `기타-옵션-view`와 `옵션1-view`를 통해 추가적인 옵션을 설정할 수 있습니다. 사용자는 `상품-디테일-이미지-그룹-view`와 `상품-착용샷-이미지-그룹-view`에서 제품의 상세 이미지와 착용샷 이미지를 동적으로 추가하고 확인할 수 있습니다.

화면 하단에는 `푸터-view`에서 시작하여 `푸터-컴포넌트-view`를 거쳐 `푸터(1)-view` 및 `푸터(2)-view`로 이어지는 푸터 컴포넌트들이 배치됩니다. 이 푸터 영역은 `푸터-왼쪽-view`, `푸터-중심-view`, `푸터-오른쪽-view`로 세분화되어 있으며, `고객센터-그룹-view`를 통해 고객 지원 정보를 제공하고, `연계-버튼-view`와 `SNS-연계-view`를 통해 외부 서비스와의 연결 기능을 제공합니다.

마지막으로, Part 1에서 전달된 `Login Form-view`의 인증 폼 데이터는 `Login Form Container-view`에 표시되어 사용자 인증을 위한 메인 컨테이너 역할을 수행합니다.

### **1. 앱 시작 및 초기 탐색**

사용자가 앱을 실행하면 먼저 **뮤즈미-인트로-screen**이 나타나 앱의 시작을 알립니다. 이후 메인 랜딩 페이지로 전환되며, 상단에는 **헤더2-touch-button**이 항상 표시됩니다. 이 버튼은 로고를 포함하고 있으며, 사용자가 언제든지 클릭하면 메인 랜딩 페이지로 돌아갈 수 있습니다.

헤더 영역에는 사용자 상태에 따라 가시성이 달라지는 버튼들이 있습니다.
*   **로그인/로그아웃-touch-button**: 사용자가 로그인하지 않은 상태에서는 "로그인" 버튼으로 표시됩니다.
*   **장바구니-touch-button** 및 **마이페이지-touch-button**: 이 버튼들은 "로그인 하지 않을땐 안보임"이라는 기능 명세에 따라, 사용자가 로그인하지 않은 초기 상태에서는 화면에 표시되지 않습니다.
*   **소개-홈페이지-touch-button**: 이 버튼은 앱의 소개 페이지로 이동하는 역할을 합니다.

### **2. 로그인 및 사용자 상태 변경**

사용자가 **로그인/로그아웃-touch-button**을 클릭하면 로그인 절차가 시작됩니다.
1.  로그인 폼이 포함된 **Login Form Container-view** (연결 정보에 명시된 컨테이너)가 나타납니다.
2.  사용자는 **Email Input-text-field**에 이메일 주소를 입력합니다. 입력된 이메일은 **Input Validator-validation**을 통해 이메일 형식 및 필수 필드 여부가 검증됩니다. (예상 데이터 타입: 이메일 형식의 문자열)
3.  **Password Input-text-field**에 비밀번호를 입력합니다. 입력된 비밀번호는 **Password Encryptor-security**를 통해 SHA-256 방식으로 해싱되어 전송 전 암호화됩니다. (예상 데이터 타입: 보안 처리된 문자열)
4.  모든 정보를 입력한 후 **Submit Button-touch-button**을 클릭하면, 검증된 이메일과 암호화된 비밀번호가 **Auth API-rest-api**로 전송되어 사용자 자격 증명을 인증합니다.
5.  인증이 성공하면 **Auth API-rest-api**의 응답(예: JWT 토큰)이 **Session Manager-storage**로 전달되어 사용자 세션 데이터가 저장됩니다.
6.  로그인 성공 후, **로그인/로그아웃-touch-button**은 "로그아웃"으로 변경되고, 이전에 숨겨져 있던 **장바구니-touch-button**과 **마이페이지-touch-button**이 화면에 표시됩니다. 사용자는 이제 마이페이지나 장바구니에 접근할 수 있습니다.

### **3. 메인 콘텐츠 탐색 및 테마/카테고리 필터링**

로그인 후 메인 페이지에서는 다양한 콘텐츠를 탐색할 수 있습니다.
1.  **배너-갤러리-display** 영역에는 여러 배너 이미지가 슬라이드 형태로 표시됩니다. 사용자는 **이전-배너-touch-button** 또는 **이후-배너-touch-button**을 클릭하여 수동으로 배너를 넘길 수 있으며, **배너-넘버링-display**는 현재 보고 있는 배너의 순서를 표시합니다.
2.  다양한 테마를 탐색하기 위한 버튼들이 제공됩니다: **전통-touch-button**, **데일리-touch-button**, **파티-touch-button**, **공주왕자-touch-button**, **아이돌-touch-button**, **국가별-touch-button**. 사용자가 이 중 하나(예: **전통-touch-button**)를 클릭하면, 해당 테마에 맞는 제품 목록이 화면에 표시됩니다.
3.  특정 테마의 전체 페이지로 이동하려면 **전통-테마-전체-페이지-입장-touch-button** 또는 **공주왕자-테마-전체-입장-touch-button**과 같은 버튼을 클릭합니다.
4.  테마 페이지 내에서는 **카테고리-샷-display**가 해당 카테고리를 대표하는 이미지를 보여주며, **테마명-display**는 현재 선택된 테마의 이름을 표시합니다.
5.  제품 목록을 세분화하기 위한 필터링 옵션도 제공됩니다. **전체-필터링-없음-touch-button**을 클릭하면 모든 필터가 해제되고 전체 상품이 표시됩니다. 특정 카테고리(예: **귀걸이-필터링-touch-button**, **목걸이-필터링-touch-button**, **반지-필터링-touch-button**, **팔찌-필터링-touch-button**, **머리장식-필터링-touch-button**, **기타-필터링-touch-button**)를 클릭하면 해당 카테고리의 제품만 필터링되어 나타납니다.
6.  필터링 결과는 **제품-갯수-요약-display**에 "총 OO개의 상품이 있습니다"와 같이 요약되어 표시됩니다.
7.  메인 페이지에는 **귀걸이-touch-button**, **목걸이-touch-button**, **반지-touch-button**, **팔찌-touch-button**, **머리장식-touch-button**, **기타-touch-button**과 같은 대분류 카테고리 버튼도 존재하며, 이들을 클릭하여 해당 카테고리의 제품 목록으로 바로 이동할 수 있습니다.

**1. 테마별 카테고리 탐색 및 갤러리 업데이트**

Part 1에서 사용자가 앱의 메인 화면 또는 카테고리 탐색 화면으로 진입하면, `각-테마별-카테고리-view`가 화면에 표시됩니다. 이 뷰는 '전통', '데일리', '공주왕자'와 같은 다양한 테마 카테고리를 시각적으로 제공하며, 각 카테고리 내에는 '귀걸이.버튼-touch-button'이나 '머리장식.버튼-touch-button'과 같은 개별 아이템 버튼들이 포함되어 있습니다 (버튼은 JSON에 명시되지 않았으나 기능 추론).

사용자가 `각-테마별-카테고리-view` 내에서 특정 테마 카테고리 버튼(예: '전통-테마-갤러리-view'에 해당하는 버튼)을 터치하면, 해당 버튼은 활성화 상태로 시각적 변화를 보입니다. 동시에 `테마별_카테고리_이미지-folder`에서 선택된 테마에 맞는 이미지 데이터가 로드되어 `전통-테마-갤러리-view` (또는 `데일리-테마-갤러리-view`, `공주왕자-테마-갤러리-view` 등 선택된 테마에 맞는 갤러리 뷰)에 제품 이미지들이 동적으로 표시됩니다. 이미지 로딩 중에는 간략한 로딩 인디케이터(추가 명세 필요)가 나타날 수 있습니다.

이 상호작용의 최종 상태는 선택된 테마의 제품 이미지들이 `전통-테마-갤러리-view` 등의 해당 갤러리 뷰에 정렬되어 표시되는 것입니다. 이 과정에서 `전통-카테고리-view`, `데일리-카테고리-view`, `공주왕자-카테고리-view`와 같은 하위 카테고리 뷰들도 함께 업데이트되어 선택된 테마의 세부 카테고리를 반영합니다.

**2. 제품 필터링 및 갤러리 결과 업데이트**

사용자가 `헤더-그룹(1)(1)-view` 또는 `헤더-그룹(1)(1)(1)-view`를 통해 제품 목록 화면으로 이동하거나, `tab-menu-view`를 통해 제품 갤러리 화면에 진입하면, `제품-그룹-view`가 화면에 나타납니다. 이 뷰는 `필터링-view`와 `제품-갤러리-view`를 포함하고 있습니다.

사용자는 `필터링-view` 내에서 가격 범위, 색상, 재질 등 다양한 필터 옵션을 선택하여 제품 목록을 세분화할 수 있습니다. 예를 들어, '가격대-touch-button'을 눌러 특정 가격 범위를 선택하거나, '색상-switch'를 활성화하여 특정 색상의 제품만 보기를 선택할 수 있습니다 (버튼/스위치는 JSON에 명시되지 않았으나 기능 추론). 필터링 옵션이 적용되면, `필터링-view`는 선택된 필터 조건을 시각적으로 표시하며, `제품-갤러리-view`는 `제품별_데이터들-folder`에서 필터 조건에 맞는 제품 데이터만 가져와 업데이트된 제품 목록을 재구성하여 보여줍니다.

이때 `제품-갤러리-요약-및-보기-옵션-view`는 필터링된 제품의 수량이나 정렬 방식을 표시하고, `제품-갤러리-4열-view`는 실제 제품 이미지와 정보를 4열 그리드 형태로 보여줍니다. 필터 옵션 선택 시 해당 옵션이 활성화되었음을 나타내는 시각적 변화(예: 버튼 색상 변경, 체크 표시)가 나타나고, `제품-갤러리-view`의 내용이 빠르게 변경됩니다. 데이터 로딩 중에는 간략한 로딩 애니메이션(추가 명세 필요)이 표시될 수 있습니다. 최종적으로 `제품-갤러리-view`는 사용자가 선택한 필터 조건에 부합하는 제품들로만 구성된 목록을 표시합니다.

**3. 제품 상세 정보 확인 및 구매/대여 옵션 설정**

`제품-갤러리-view`에서 사용자가 특정 제품 이미지를 터치하면, 해당 제품의 상세 페이지로 이동합니다. 이 페이지에서는 `Untitled(1)-view` 컨테이너 내에 `구매-정보-view`가 표시됩니다. `구매-정보-view`는 `제품별_데이터들-folder`에서 가져온 해당 제품의 상세 정보를 바탕으로 `금액-소개-view`, `구먀/대여-결정-여부-view`, `기타-옵션-view` 등의 하위 뷰들을 채웁니다.

사용자는 `구먀/대여-결정-여부-view` 내에서 '구매' 또는 '대여' 옵션을 선택할 수 있습니다 (예: '구매-touch-button', '대여-touch-button' 또는 '구매/대여-switch' - JSON에 명시되지 않았으나 기능 추론). 이 선택에 따라 `금액-소개-view`에 표시되는 가격 정보가 구매가 또는 대여가로 즉시 변경됩니다. 또한, `기타-옵션-view` 내의 `옵션1-view`와 같은 추가 옵션들이 활성화되거나 비활성화될 수 있으며, 선택 가능한 옵션의 종류가 변경될 수 있습니다. 예를 들어, '대여' 선택 시 '대여 기간-text-field'가 활성화될 수 있습니다 (추가 명세 필요).

`상품-디테일-이미지-그룹-view`와 `상품-착용샷-이미지-그룹-view`는 해당 제품의 상세 이미지와 착용샷 이미지를 동적으로 표시하며, "사용자가 이미지를 추가한 만큼 넣게 할 것"이라는 기능에 따라 유닛들이 확장되어 보여집니다. '구매' 또는 '대여' 옵션 선택 시 해당 선택이 시각적으로 강조되며, `금액-소개-view`의 텍스트가 즉시 업데이트됩니다. `기타-옵션-view` 내의 하위 옵션들의 활성화/비활성화 상태가 명확하게 표시됩니다. 최종적으로 사용자가 선택한 구매/대여 방식과 추가 옵션에 따라 제품 정보가 최종적으로 구성되며, 결제 또는 다음 단계로 진행할 준비가 완료됩니다.

**4. 로그인 폼 상호작용 (별도 흐름)**

사용자가 앱 내에서 로그인 또는 인증이 필요한 기능에 접근하거나, 앱 시작 시 로그인 화면으로 진입합니다 (예: Part 1에서 '로그인-touch-button' 클릭 시). 이 경우 `Login Form Container-view`가 화면에 표시됩니다. 이 뷰는 사용자 인증을 위한 입력 필드(예: '아이디-text-field', '비밀번호-text-field' - JSON에 명시되지 않았으나 기능 추론)와 '로그인-touch-button'을 포함할 것으로 예상됩니다.

`제품별_데이터-data` (미정의된 데이터 소스)가 `Login Form Container-view`로 연결되어 있으나, 이 연결의 구체적인 역할은 추가 명세가 필요합니다. 추론컨대, 로그인 폼에 특정 제품 관련 정보가 미리 채워지거나, 로그인 후 개인화된 제품 추천을 위한 데이터로 활용될 수 있습니다. 사용자가 입력 필드에 정보를 입력하면 실시간 유효성 검사 피드백(예: 잘못된 형식의 이메일 주소 입력 시 경고 메시지 - 추가 명세 필요)이 표시될 수 있습니다. '로그인-touch-button' 클릭 시, 인증 처리 중임을 나타내는 로딩 인디케이터가 표시될 수 있습니다. 로그인 성공 시 사용자는 다음 화면으로 이동하거나 이전에 접근하려던 기능으로 돌아가며, 로그인 실패 시 오류 메시지가 표시되고, 사용자는 다시 정보를 입력하거나 '비밀번호 찾기-touch-button'과 같은 다른 옵션을 선택할 수 있습니다.
